SpringBoot 中间件设计和开发

第 9 章 结合 SpringBoot 开发 ORM Starter

一、前言

开发一个中间件你会遇到什么样的技术障碍？

有些时候，有些知识，不是因为难不会，而是因为不知道而不会。这类的技术点并不会多难，因为这些知识都不是对复杂数学逻辑的具体实现过程学习，只是一种约定、规范、标准而已。

就像如果你没尝试过把业务系统中的非业务逻辑开发成组件，那么可能就不会想到要这么做。当后来你了解了有中间件这样一种解决方案时，可能你会遇到不知道中间件开发的框架是什么样，该怎么读取一些定义的配置或者是注解，以及如何与 SpringBoot 结合使用。

这些问题在你实际开发过程中都会一点点遇到，其实最好的学习方式的是有类似的成长土壤或者系统的学习过，但这方面资料其实并不多，它不像业务开发有很多的资料可以查阅。所以如果以上都没有，又遇到的是新问题，那么阅读同类的源码是最好的方式。

二、需求背景

我们已经在前两章实现了 ORM 框架，也把自己实现的 ORM 框架与 Spring 结合，可以让我们像使用 MyBatis-Spring 一样使用我们的实现的组件。

那么接下来我们需要做的就是，把这个自己实现的 mybatis 和 mybatis-spring，与 SpringBoot 结合，开发出一个类似 mybatis-spring-boot-starter 的 ORM 组件。

mybatis-spring-boot-starter 的官方源码在这里： github.com/mybatis/spr… 感兴趣的小伙伴可以阅读一下。

三、方案设计

现在我们手里有了自己实现的 ORM 框架和 mybatis-spring，这两个组件已经可以满足我们像使用其他 ORM 框架一样的方式在系统服务中使用。有了这两个基础的组件就可以结合 SpringBoot 开发一个自己的 MyBatis SpringBoot Starter。

在 SpringBoot 的 Starter 开发中，主要的核心在于自动加载 AutoConfiguration 的使用，这里我们需要加载 yml 或者其他的配置文件，并结合配置信息扫描注册相关的 Bean 类信息，以及初始化相关的组件启动对象。

整体的设计方案如图 9-1 MyBatis SpringBoot Starter 中间件设计。

  • 左侧 ORM、mybatis-spring，是我们已经实现的好组件功能，右侧的 SpringBoot Starter 是本次需要开发的内容。
  • 结合 SpringBoot 开发 ORM 的 Starter 与我们使用 mybatis-spring 是有差别的，主要在于加载配置和启动方式，所以这里需要更改一些我们实现的组件 mybatis、mybatis-spring 中的一些扫描接口、注册Bean的代码。
  • 为了让大家方便的学习这部分内容，不让知识之间混淆，我们把自己实现的 mybatis、mybatis-spring 代码都整合到本章节需要开发的这个 ORM SpringBoot Starter 中，也就是这个工程名为 mybatis-spring-boot-starter。
  • 另外需要注意官网的 mybatis starter 是通过引入其他两个组件进行开发的，如果阅读源码可以关注下。

mybatis-spring-boot-starter 中间件类关系，如图 9-2

在整个中间件实现工程中分为三块，autoconfigure、mybatis、spring，它们的作用分别如下：

  • autoconfigure，读取自定义配置信息以及负责把相关 mybatis、spring 中的 Bean 加载启动。
  • mybatis，这部分与我们之前实现的 ORM 框架源码几乎是一致的，只是对 SqlSessionFactoryBuilder 做了符合 yml 配置方式的加载处理。
  • spring，这一部分主要的变动是 MapperScannerConfigurer 关于扫描定义 Bean 信息时 addGenericArgumentValue 入参信息的变更。

2. mybatis SqlSessionFactoryBuilder 构建类修改

cn.bugstack.middleware.mybatis.boot.mybatis.SqlSessionFactoryBuilder.java

public class SqlSessionFactoryBuilder {

    public DefaultSqlSessionFactory build(Connection connection, String packageSearchPath) throws Exception {
        Configuration configuration = new Configuration();
        configuration.setConnection(connection);
        // 读取配置
        ResourcePatternResolver resourcePatternResolver = new PathMatchingResourcePatternResolver();
        Resource[] resources = resourcePatternResolver.getResources(packageSearchPath);
        List<Element> list = new ArrayList<>(resources.length);
        for (Resource resource : resources) {
            Document document = new SAXReader().read(new InputSource(new InputStreamReader(resource.getInputStream())));
            list.add(document.getRootElement());
        }
        configuration.setMapperElement(mapperElement(list));
        return new DefaultSqlSessionFactory(configuration);
    }
    
    // ...
}

在 ORM 中间件实现的章节我们有详细介绍过此类的功能，它是一个构建实例化元素、解析配置、链接数据库、解析sql语句等操作的类，这里我们把构建实例化元素的方法做了一些调整。

  • 首先是 build 传参信息提供了数据库实例 Connection 和 packageSearchPath 的扫描路径。
  • 其实 packageSearchPath 就是 classpath*:mapper/*.xml 的路径信息，这不过这里的处理方式是交给 ResourcePatternResolver 类处理，获取 InputStream。
  • 接下来的处理内容就与 ORM 原有的内容一致了，其实这样的代码是可以结合到我们的 ORM 框架中，这里引入组件包就可以，但为了让读者更加方便，我们把整个自己实现的 ORM 框架代码复制了过来。

3. spring MapperFactoryBean 入参信息改变

cn.bugstack.middleware.mybatis.boot.spring.MapperFactoryBean.java

public class MapperFactoryBean<T> implements FactoryBean<T> {

    private Logger logger = LoggerFactory.getLogger(MapperFactoryBean.class);

    private Class<T> mapperInterface;
    @Resource
    private SqlSessionFactory sqlSessionFactory;

    public MapperFactoryBean(Class<T> mapperInterface) {
        this.mapperInterface = mapperInterface;
    }
    
    // ...
}
  • 其实这个类变化的信息并不多，如果你还记得 ORM 与 Spring 结合的代码，可以想起 MapperFactoryBean 构造函数中的入参信息只有一个 mapperInterface 了。
  • 这里我们把 sqlSessionFactory 从构造函数的入参中拿了出来，通过 @Resource 注解来注入。这是因为 SpringBoot Starter 的加载机制，在扫描 Bean 初始化时候，并不会实例化 sqlSessionFactory，此时传参只是一个 NULL，所有我们把这个入参改为注解注入的方式。

4. autoconfigure 信息处理

定义配置 Yml 配置信息

cn.bugstack.middleware.mybatis.boot.autoconfigure.MybatisProperties

@ConfigurationProperties(prefix = MybatisProperties.MYBATIS_PREFIX)
public class MybatisProperties {

    public static final String MYBATIS_PREFIX = "mybatis.datasource";

    private String driver;              // com.mysql.jdbc.Driver
    private String url;                 // jdbc:mysql://127.0.0.1:3306/bugstack?useUnicode=true
    private String username;            // root
    private String password;            // 123456
    private String mapperLocations;    // classpath*:mapper/*.xml
    private String baseDaoPackage;      // cn.bugstack.middleware.test.infrastructure.dao
    
    //...get/set
}
  • @ConfigurationProperties，用于创建指定前缀( prefix = MybatisProperties.MYBATIS_PREFIX )的自定义配置信息，这样就在 yml 或者 properties 中读取到我们自己设定的配置信息。
  • 配置信息：driver、url、username、password、mapperLocations、baseDaoPackage，这些只要在 SpringBoot 中使用过 MyBatis 就比较熟悉了。

加载配置和初始化 Bean

cn.bugstack.middleware.mybatis.boot.autoconfigure.MybatisAutoConfiguration.java

@Configuration
@ConditionalOnClass({SqlSessionFactory.class})
@EnableConfigurationProperties(MybatisProperties.class)
public class MybatisAutoConfiguration implements InitializingBean {

    @Bean
    @ConditionalOnMissingBean
    public SqlSessionFactory sqlSessionFactory(Connection connection, MybatisProperties mybatisProperties) throws Exception {
        return new SqlSessionFactoryBuilder().build(connection, mybatisProperties.getMapperLocations());
    }

    @Bean
    @ConditionalOnMissingBean
    public Connection connection(MybatisProperties mybatisProperties) {
        try {
            Class.forName(mybatisProperties.getDriver());
            return DriverManager.getConnection(mybatisProperties.getUrl(), mybatisProperties.getUsername(), mybatisProperties.getPassword());
        } catch (ClassNotFoundException | SQLException e) {
            e.printStackTrace();
        }
        return null;
    }

    public static class AutoConfiguredMapperScannerRegistrar implements EnvironmentAware, ImportBeanDefinitionRegistrar {

        private String basePackage;

        @Override
        public void registerBeanDefinitions(AnnotationMetadata importingClassMetadata, BeanDefinitionRegistry registry) {
            BeanDefinitionBuilder builder = BeanDefinitionBuilder.genericBeanDefinition(MapperScannerConfigurer.class);
            builder.addPropertyValue("basePackage", basePackage);
            registry.registerBeanDefinition(MapperScannerConfigurer.class.getName(), builder.getBeanDefinition());
        }

        @Override
        public void setEnvironment(Environment environment) {
            this.basePackage = environment.getProperty("mybatis.datasource.base-dao-package");
        }
    }

    @org.springframework.context.annotation.Configuration
    @Import(AutoConfiguredMapperScannerRegistrar.class)
    @ConditionalOnMissingBean({MapperFactoryBean.class, MapperScannerConfigurer.class})
    public static class MapperScannerRegistrarNotFoundConfiguration implements InitializingBean {

        @Override
        public void afterPropertiesSet() {
        }

    }

    @Override
    public void afterPropertiesSet() throws Exception {
    }

}

以上这段代码看上去比较多，但整个实现的内容并不复杂，主要包括了实例化两个 Bean(connection、sqlSessionFactory) 和处理自动扫描注册 Bean 信息到 Spring 容器中。

  • @EnableConfigurationProperties(MybatisProperties.class)，用于引入配置信息，如果你不引入是不能使用配置信息的。
  • Connection connection(MybatisProperties mybatisProperties)，实例化数据库连接，mybatisProperties 是这个方法的入参，这里也可以通过属性注入方法体里使用的方式进行处理。另外，这里返回的数据库连接方式，是我们从类 SqlSessionFactoryBuilder 提取出来的，这样的方式也非常符合官网中 mybaits SpringBoot Starter 的处理方式，因为这样可以更加方便的增加其它的数据库连接池。
  • SqlSessionFactory sqlSessionFactory(Connection connection, MybatisProperties mybatisProperties)，实例化 SqlSession 的链接工厂，首先可以看到入参信息包括了我们的这里创建的 connection，另外还有配置入参信息。
  • 最后是内部类 AutoConfiguredMapperScannerRegistrar 的实现，注册 Bean 信息是优先于 Bean 实例化的，所以这里需要通过实现 EnvironmentAware 读取 yml 中的配置信息，因为这个配置信息是用于扫描 DAO 接口类，注册 Bean 的。

引入配置和启动

@Configuration
@Import(AutoConfiguredMapperScannerRegistrar.class)
@ConditionalOnMissingBean({MapperFactoryBean.class, MapperScannerConfigurer.class})
public static class MapperScannerRegistrarNotFoundConfiguration implements InitializingBean {
    @Override
    public void afterPropertiesSet() {
    }
}    

@Override
public void afterPropertiesSet() throws Exception {
}
  • 在 MybatisAutoConfiguration 类中，还有一个内部类 MapperScannerRegistrarNotFoundConfiguration，它的作用是把我们的用于扫描加载配置和初始化的类启动起来。
  • 其实它的方式也非常简单，主要通过引入 @Import(AutoConfiguredMapperScannerRegistrar.class) 在实现了 InitializingBean 类上，就可以被 Spring 处理了。

5. 启动自动配置

resources/META-INF/spring.factories

org.springframework.boot.autoconfigure.EnableAutoConfiguration=cn.bugstack.middleware.mybatis.boot.autoconfigure.MybatisAutoConfiguration
  • @EnableAutoConfiguration 的作用是从 classpath 中搜索所有 META-INF/spring.factories 配置文件然后将其中 org.springframework.boot.autoconfigure.EnableAutoConfiguration 配置的信息加载到 spring 容器。

六、总结

  • 本章节的学习需要注意几个知识点：AutoConfiguration 的使用以及它的初始化顺序、自定义配置的指定读取、Bean 信息的注入创建，以及了解整个中间件框架的开发方式。
  • 关于以上中间件的实现更多的是会把最核心的流程和实现内容展示给大家，但像一些额外流程的处理是不足的，例如一些异常、特殊情况、不同查询类型等等，这些大家在学习了整体的功能后，是可以不断补充进去的，这些实现并不会太复杂，但对于学习者确实一个很好的实现和学习过程。
  • 中间件的设计可以解决很多重复、复杂的功能实现，既可以把这样的技术深入使用，也可以在同类服务中纵向体现，积极的运用这些技术才能更熟练的解决业务问题。
